> 数据来源：48个电气故障案例（统计截止2026-07-14）。涉及[[entities/EHPS转向机]]（20例）、[[entities/RAS助力缸]]（27例）、HPS转向机（1例，EPS标定失败）。

## 分类详述

### 1. 标定/参数配置问题（10例）

**EHPS（6例）**：最常见电控问题模式。涉及更换转向机后需要标定（#10）、多台车无法标定需要多次尝试（#112）、标定未按正确流程操作导致失败（#25）、软限位标定不当触发液压失效报警（#138）、未标定导致外部控制LOCK（#107、#177）。多例证实"仅上15电、不启动发动机"标定才能成功写入。

**RAS（3例）**：刷写配置参数时工具不当导致针脚损坏（#228）、软件版本与目的地不匹配（#60）、更换传感器后需重新标定（#111）。

**HPS（1例）**：EPS标定失败实际由线束无电导致（#184），体现标定失败不一定都是标定流程问题。

> 典型模式：标定显示成功但实际未写入 → 确认是否在启动车辆状态下标定，应仅上15电。

### 2. CAN通讯/信号故障（9例）

**全部为RAS案例（9例）**。这是RAS产品最突出的故障模式。

具体表现包括：
- CAN信号丢帧/超时：SAS-EBS信号无效（#93）、方向盘角度CAN超时（#223）、车轮信号错误/丢帧（#193）
- 整车CAN信号缺失：缺少VDC2/EEC1等必要信号（#66）
- 信号源地址错误：转向机信号未转发给仪表导致误报灯（#173）
- 车载信号质量：里程信号错误（#45）、历史CAN故障消除后偶发亮灯（#220、#253）
- 多车群体性问题：挪威订单多台同时报CAN相关故障（#53、#220、#223、#253）

**这些案例中博世产品端未见异常，均为整车CAN信号质量问题。**

### 3. 线束/接插件问题（8例）

**EHPS（4例）**：整车轮线问题导致软件报错（#19），整车线插口虚接导致MC/CRC错误（#21），事故车线束损伤导致多DTC（#26），CAN线到转向机端虚接/短路/针脚腐蚀（#189）。

**RAS（4例）**：X4端口针脚变形虚接导致传感器初始化超时（#108、#140），整车线路未搭铁导致RAS无法通讯（#128），X1线束进水导致RAS不工作（#51）。

> 核心排查点：**X4端口针脚（传感器线束）和X3端口15电针脚**是RAS产品最常见的线束故障点。

### 4. 智驾/外部控制通讯异常（5例）

**全部为EHPS案例（5例）**。这是EHPS产品特有的高端故障模式，与智驾系统交互相关。

- MC信号异常导致转向进LOCK退出智驾（#127）
- 上游EPSI1节点发送异常导致LOCK（#160）
- AD请求时机不当（上电立即请求，助力未就绪）导致Cancel（#167）
- 颠簸路面偶发LOCK（#178）
- 整车退出智驾，涉及EPS 2级故障（#52）

> 根因多为整车智驾系统与转向机控制时序不匹配，而非转向机自身故障。

### 5. 电机/泵体机械故障（5例）

**EHPS（1例）**：电机不工作，更换转向机后解决（#192），可能为电机内部故障。

**RAS（4例）**：联轴器损坏→泵卡滞（#183），泵干磨导致短暂工作后失效（#218），更换动力单元解决（#202），安全校验DTC更换动力单元后消失（#229）。

> RAS动力单元内部泵体/联轴器失效是RAS不工作的潜在硬件根因之一，尤其当电气排查全部正常时需考虑。

### 6. 传感器/角度信号异常（4例）

**EHPS（1例）**：TAS（转向角传感器）数值无效导致亮黄灯（#121）。

**RAS（3例）**：后桥传感器初始化超时（#79），VDHR信号问题导致后桥传感器校验报错（#80），转角信号错误引发事故（#89）。

### 7. 供电/保险问题（2例）

- EHPS：保险烧断导致无电助力（#4）
- RAS：电源线正负极接反/插头错误导致烧保险（#101）

> "先看保险"在EHPS和RAS上都适用。

### 8. 系统排气/油液问题（2例）

**全部为RAS案例**：
- 更换助力缸后油液加注方式不当导致系统有气（#88）
- 加注时系统内有气导致控制延迟，需摆动轮胎排气（#233）

> RAS液压系统需使用正确加注方式（手动模式、适当提高泵转速），不能使用自动加注机。

### 9. 软件/配置问题（2例）

**全部为EHPS案例**：
- 更新转向机最新版软件（#165）
- 转子角无效故障，台架复现后确认需软件修复（#176）

### 10. 整车信号/配置逻辑问题（1例）

RAS案例（#56）：客户反映RAS不工作，实际原因为配置了公交站模式（停车开门后RAS居中），属正常设计行为而非故障。

## EHPS vs RAS 故障模式对比

| 故障模式 | EHPS（20例） | RAS（27例） |
|---------|:----------:|:----------:|
| 标定/参数配置问题 | 6（30.0%） | 3（11.1%） |
| 智驾/外部控制通讯异常 | 5（25.0%） | 0 |
| 线束/接插件问题 | 4（20.0%） | 4（14.8%） |
| 软件/配置问题 | 2（10.0%） | 0 |
| CAN通讯/信号故障 | 0 | 9（33.3%） |
| 电机/泵体机械故障 | 1（5.0%） | 4（14.8%） |
| 传感器/角度信号异常 | 1（5.0%） | 3（11.1%） |
| 供电/保险问题 | 1（5.0%） | 1（3.7%） |
| 系统排气/油液问题 | 0 | 2（7.4%） |
| 整车信号/配置逻辑问题 | 0 | 1（3.7%） |

**关键差异点：**

1. **EHPS更突出三大类**：标定（30%）> 智驾交互（25%）> 线束（20%），合计占75%
2. **RAS更突出通讯类**：CAN通讯/信号故障（33.3%）远超其他模式，其次为线束和泵体机械（各约15%）
3. **智驾交互类故障仅存在于EHPS**：这是EHPS与智驾系统深度耦合导致的特殊性
4. **CAN通讯故障仅存在于RAS**：RAS依赖整车CAN信号输入较多，整车信号质量问题突出
5. **泵体机械故障RAS明显多于EHPS**：RAS动力单元内部联轴器和泵体为关键失效点

## 关键知识点

- EHPS最突出：**标定是常见电控问题**，很多场景重新标定可解决
- RAS最突出：**CAN通讯/信号故障**远高于其他模式，常见于海外订单
- 电液转向机不工作先查保险
- 整车退出智驾模式通常是转向机报LOCK/异常信号触发的保护机制
- RAS被误报为不工作时不一定是故障，需排除公交站模式等配置逻辑
